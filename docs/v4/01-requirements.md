# V4 需求文档：自动回复评论

## 功能概述

新增"自动回复评论"模式，自动监控并回复自己微博下收到的评论，包括：
- **直接评论**：别人在自己微博下的评论
- **楼中楼回复**：别人回复自己已发出的评论（评论的评论）

## 核心需求

### 1. 评论监控
- 定时轮询获取自己收到的新评论
- 支持两种来源：
  - `comments/to_me.json`：获取收到的所有评论（包括楼中楼）
  - 备选：遍历自己的微博，通过 `comments/show.json` 获取评论列表
- 过滤已回复的评论，避免重复回复
- 支持配置每日回复上限

### 2. 上下文收集
回复评论时，需要收集完整上下文供AI生成高质量回复：
- **微博原文内容**：被评论的微博正文
- **评论内容**：需要回复的评论文本
- **父评论内容**（楼中楼场景）：如果是回复评论的评论，需要包含被回复的那条评论

### 3. AI回复生成
- 复用现有 `ai_generator.py` 的LLM调用能力
- 新增专用的回复 prompt 模板（区别于主动评论的风格）
- 回复风格要求：
  - 针对性回复，不是泛泛而谈
  - 根据评论语气匹配回复风格
  - 保持口语化、自然、有人味
  - 作为微博博主的身份回复

### 4. 回复发送
- 通过 `comments/reply.json` API 发送回复
- 回复需要指定：目标微博ID、目标评论ID、回复内容
- 支持配置回复间隔，模拟人类操作节奏

## 约束条件

1. **不回复自己**：跳过自己发出的评论
2. **不重复回复**：已回复过的评论不再回复
3. **每日上限**：可配置每日回复数量上限
4. **工作时段**：复用现有工作时段配置
5. **频率控制**：回复间隔随机化，避免被反爬
6. **API限制**：微博API有调用频率限制，需合理控制轮询频率

## 配置需求

在 `config.yaml` 中新增 `reply` 配置段：
```yaml
reply:
  enabled: true
  daily_limit: 30
  poll_min: 60        # 轮询间隔（秒）
  poll_max: 180
  reply_delay_min: 5  # 回复前延迟（秒）
  reply_delay_max: 15
  prompt: "weibo_reply"  # 使用的prompt模板名
```

在 `prompts.yaml` 中新增 `weibo_reply` 模板，适配回复场景。
