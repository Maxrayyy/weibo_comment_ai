# V3 需求文档：表情支持 + 图片识别

## 功能一：表情读取与发送

### 需求描述
- 从微博API获取官方表情列表，缓存到本地
- 在AI生成评论时，允许模型使用微博表情（如 `[哈哈]`、`[泪]`）
- 发布评论时表情以 `[表情名]` 格式嵌入文本，微博会自动渲染为表情图标

### 约束
- 表情名必须精确匹配微博官方列表，否则会显示为纯文本
- 不是每条评论都需要表情，由模型自然决定
- 评论长度限制（140字）计算时，`[哈哈]` 按4个字符计算

## 功能二：图片识别辅助评论

### 需求描述
- 抓取微博时同时提取第一张图片URL
- 生成评论时，将图片传给多模态模型，让模型结合文字+图片生成更有针对性的评论
- 仅取第一张图片（降低API成本和延迟）

### 约束
- 图片为可选项：无图微博仍走纯文本生成
- 需要多模态模型支持（当前用的qwen系列支持vision）
- 图片URL使用中等尺寸（bmiddle/mw690），兼顾清晰度和传输速度
- 三种抓取模式（API/好友圈Selenium/超话Selenium）都需要支持图片提取
